## 2. How would you track down a performance issue in production? Have you ever had to do this?

I usually start by identifying whether the issue is coming from the frontend, backend, database, or network requests.

For frontend issues, I use Chrome DevTools, Lighthouse, and Network tabs to check rendering time, API delays, large assets, or unnecessary re-renders.

For backend-related issues, I check logs, database queries, response times, and try to reproduce the issue locally.

In previous work, I have handled performance-related issues like:

* reducing unnecessary API calls
* optimizing component rendering
* compressing assets/images
* improving page load times by restructuring frontend logic

I believe breaking the problem into smaller parts and checking one layer at a time helps find the root cause faster.
